# 설곽 졸업사진용 일회성 프로그램 계획서

## 1. 프로그램 개요

### 1-1. 프로그램명
- **설곽 졸업사진 개인별 열람 프로그램**

### 1-2. 개발 목적
졸업사진 촬영 이후에는 대량의 사진이 한꺼번에 생성되어, 운영자가 인물별로 분류해 전달하는 데 많은 시간과 노력이 필요합니다. 또한 학생 입장에서도 전체 사진 중 본인이 나온 사진만 선별하기 어렵습니다.

이에 따라, 졸업사진을 하나의 공간에 업로드한 뒤 **얼굴 인식 기반 자동 분류**를 통해 학생들이 본인이 포함된 사진을 빠르게 열람할 수 있는 내부용 일회성 프로그램을 제작합니다.

### 1-3. 사용 대상
- 서울과학고 졸업사진 촬영 참여 학생
- 사진 업로드 담당자
- 프로그램 관리자(소수)

### 1-4. 사용 범위
- 설곽 졸업사진 행사 한정
- 외부 공개 없이 내부 인원만 사용
- 행사 종료 후 필요 시 데이터 삭제

---

## 2. 개발 배경 및 필요성

졸업사진 촬영에서는 다양한 장소와 구도로 많은 사진이 촬영됩니다. 이를 메신저나 일반 클라우드로 단순 공유할 경우 다음 문제가 발생합니다.

- 사진 수가 많아 원하는 사진을 찾기 어렵다.
- 인물별로 일일이 골라 전달하는 과정이 번거롭다.
- 단체 폴더에서는 본인이 나온 사진을 빠르게 확인하기 어렵다.

기존 공유 방식은 저장과 열람에는 적합하지만, 특정 인물 기준 자동 정리 기능은 부족합니다. 본 프로그램은 이러한 갭을 보완하여, **졸업사진이라는 특정 상황에 최적화된 열람 경험**을 제공합니다.

---

## 3. 개발 목표

### 3-1. 핵심 목표
- 여러 장의 졸업사진을 한곳에 업로드할 수 있다.
- 사진 속 얼굴을 인식해 유사한 얼굴끼리 그룹화할 수 있다.
- 학생이 자신의 얼굴을 기준으로 본인이 포함된 사진을 확인할 수 있다.

### 3-2. 세부 목표
- 업로드 과정을 단순화한다.
- 관리자 입장의 배포 부담을 줄인다.
- 학생의 사진 탐색 시간을 단축한다.
- 일회성 운영에 맞춰 최소 기능으로 구현한다.

---

## 4. 주요 기능

### 4-1. 사진 업로드
- 관리자 또는 지정 인원이 졸업사진 파일 업로드
- 다중 파일 일괄 등록 지원
- 업로드 파일을 서버 또는 지정 폴더에 저장

### 4-2. 얼굴 검출
- 각 사진에서 얼굴 위치 자동 탐지
- 단체사진 포함 다중 얼굴 개별 추출

### 4-3. 얼굴 그룹화
- 얼굴 특징값(임베딩) 비교 기반 유사 얼굴 군집화
- 동일 학생으로 추정되는 얼굴을 하나의 그룹으로 정리

### 4-4. 개인별 사진 열람
- 사용자는 자신의 얼굴 기준으로 관련 사진 목록 조회
- 전체 사진 탐색 없이 주요 사진을 빠르게 확인

### 4-5. 관리자 보정
- 오분류된 사진의 그룹 이동/병합/분리
- 자동 분류 결과를 수동으로 보완

---

## 5. 개발 방식

### 5-1. 운영 형태
본 프로그램은 장기 서비스가 아닌 **내부용 일회성 도구**로 운영합니다.

따라서 회원가입, 대규모 인프라, 상용 배포를 우선순위에서 제외하고 실제 사용에 필요한 핵심 기능 중심으로 개발합니다.

### 5-2. 기본 처리 흐름
1. 졸업사진 업로드
2. 얼굴 자동 탐지
3. 얼굴 특징 추출 및 유사도 비교
4. 인물별 그룹 생성
5. 사용자별 사진 열람

### 5-3. 예상 기술 요소
- 프론트엔드: 간단한 웹 페이지
- 백엔드: Python 기반 서버
- 얼굴 인식: 공개 라이브러리 활용
- 저장소: 로컬 디스크 또는 간단한 서버 스토리지

---

## 6. 기대 효과

### 6-1. 사용자 측면
- 본인 포함 사진을 빠르게 탐색 가능
- 불필요한 수동 탐색 피로 감소
- 행사 사진 열람 경험 개선

### 6-2. 운영 측면
- 사진 전달/정리 업무 부담 완화
- 인물별 수작업 분류 최소화
- 행사 후 데이터 정리 체계화

### 6-3. 기술적 측면
- 얼굴 인식 기술의 실제 적용 경험 확보
- 내부 도구 개발 경험 축적
- 향후 타 행사 응용 가능성 검증

---

## 7. 예상 문제점 및 보완 방안

### 7-1. 얼굴 인식 오류
조명, 각도, 표정, 해상도에 따라 인식 정확도가 저하될 수 있음.

**보완 방안**
- 관리자 보정 UI 제공
- 완전 자동 분류가 아닌 “후보 추천” 개념으로 운영

### 7-2. 개인정보 이슈
학생 얼굴 정보는 민감 정보로 취급될 수 있음.

**보완 방안**
- 외부 공개 금지, 내부 인원 한정 사용
- 행사 종료 후 사진/임베딩 데이터 삭제 가능 구조
- 최소 인원에게만 관리자 권한 부여

### 7-3. 일회성 대비 과도한 개발
초기 범위를 넓히면 일정·품질 리스크 증가.

**보완 방안**
- MVP 중심 우선 개발
- 로그인/정교 권한/다운로드 제어는 필요 시 후속 추가

본 프로그램은 설곽 졸업사진 공유/열람 효율을 높이기 위한 내부용 일회성 도구입니다.

기존의 단순 클라우드 공유 방식과 달리 얼굴 인식 기반 분류를 적용하여 학생들이 본인 포함 사진을 더 빠르게 확인할 수 있습니다. 또한 외부 공개를 전제로 하지 않기 때문에 개인정보 리스크와 개발 부담을 낮추면서도 실사용 가능성을 검증할 수 있습니다.

따라서 본 계획은 기술적으로 실현 가능하며, 행사 사진 관리의 비효율을 실질적으로 줄일 수 있는 현실적인 프로젝트 제안입니다.
